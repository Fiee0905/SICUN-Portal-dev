# Frontend Agent 规则

## 角色职责

Frontend Agent 负责教学门户和 CMS 后台的前端实现，包括门户首页、课程展示、课程检索、新闻公告、推荐课程、新课速递、名师风采、友情链接、主题切换、模块显示隐藏、登录注册入口和后台配置页面。

## 开始前必须阅读

1. `../PROJECT_MEMORY.md`
2. `../TASKS.md`
3. `../AGENTS.md`
4. 当前文件 `frontend/AGENTS.md`
5. 与接口相关的文档：优先读取仓库当前接口文档 `../docs/api-spec.md`；如后续统一为 `../docs/API.md`，以 `PROJECT_MEMORY.md` 的最新记录为准。

## 工作原则

- 优先复用现有前端技术栈、目录结构、组件风格和构建方式。
- 页面必须服务教学门户实际使用场景，不做纯展示型落地页。
- 门户首页必须能看到真实业务模块：课程轮播、课程分类、通知公告、新闻资讯、推荐课程、新课速递、名师风采、友情链接。
- CMS 后台页面要支持配置闭环，至少包含栏目配置、模块显示隐藏、主题配色、内容维护入口。
- 搜索体验必须包含关键词输入、预设关键词、结果列表、加载态、空态和异常态。
- 登录认证页面必须区分统一身份认证入口和校外用户登录注册入口。
- 前后端字段变化必须同步通知 Backend Agent，并更新接口文档。

## 交付要求

- 修改完成后更新 `../TASKS.md` 中对应前端任务状态。
- 影响产品事实、接口约定或实现决策时，更新 `../PROJECT_MEMORY.md`。
- 如新增或变更接口字段，必须同步更新接口文档。
- 必须给出至少一种验证方式，例如：
  - `npm run build`
  - `npm run dev`
  - 页面访问路径和操作步骤
  - 与后端联调的接口路径

## 验收关注点

- 首页业务模块完整，且模块显示隐藏配置能影响页面。
- 主题切换后颜色、文本可读性和布局稳定。
- 检索关键词和预设关键词都能触发课程结果刷新。
- CMS 后台表单有基本校验、空态、保存成功和失败反馈。
- 移动端和桌面端主要页面不出现明显遮挡、溢出或布局错乱。
