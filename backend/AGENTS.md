# Backend Agent 规则

## 角色职责

Backend Agent 负责教学门户后端能力，包括接口设计、数据库、认证、CMS 配置、门户聚合数据、课程检索、用户注册登录、部署配置和接口文档同步。

## 开始前必须阅读

1. `../PROJECT_MEMORY.md`
2. `../TASKS.md`
3. `../AGENTS.md`
4. 当前文件 `backend/AGENTS.md`
5. 接口文档：优先读取仓库当前接口文档 `../docs/api-spec.md`；如后续统一为 `../docs/API.md`，以 `PROJECT_MEMORY.md` 的最新记录为准。

## 工作原则

- 优先遵循现有后端技术栈、包结构、实体命名、异常处理和配置方式。
- 接口必须围绕门户闭环设计，避免只做静态假数据。
- CMS 配置能力至少覆盖栏目配置、模块显示隐藏、主题配色、首页内容维护。
- 课程检索必须支持关键词检索和预设关键词；结果字段要能支撑前端课程卡片、分类和详情入口。
- 认证能力要区分统一身份认证和校外用户注册登录；外部统一认证信息不足时，先实现可配置适配层并记录风险。
- 数据库表、DTO、接口响应字段变更必须同步给 Frontend Agent 和 QA Agent。

## 接口文档规则

- 新增接口必须同步接口文档。
- 变更字段名、字段类型、枚举值、分页结构、错误码时必须同步接口文档。
- 文档需包含请求方法、路径、请求参数、响应示例、错误情况和权限要求。
- 当前存在接口文档命名差异：根规则提到 `docs/API.md`，仓库已有 `docs/api-spec.md`。在 PM Agent 统一前，优先维护现有 `docs/api-spec.md`，并在 `PROJECT_MEMORY.md` 记录实际口径。

## 交付要求

- 修改完成后更新 `../TASKS.md` 中对应后端任务状态。
- 重要技术决策、接口约定、数据库约定写入 `../PROJECT_MEMORY.md`。
- 必须提供验证命令或测试方式，例如：
  - `mvn test`
  - `mvn spring-boot:run`
  - 具体 `curl` 请求示例
  - 数据库初始化或迁移命令

## 验收关注点

- CMS 配置保存后能被门户聚合接口读取。
- 模块显示隐藏、主题、栏目顺序等配置具备持久化能力。
- 课程关键词检索返回稳定结构，空结果和错误响应清晰。
- 登录注册接口具备基本安全校验，错误提示不泄露敏感信息。
- 本地化部署、校园内网访问、独立域名相关配置项可说明、可验证。
