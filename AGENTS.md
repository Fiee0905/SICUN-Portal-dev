# 项目总说明

本项目是四川师范大学课程教学门户/课程资源平台，需满足投标文件技术方案中的功能要求。

## 固定分工

- PM Agent：负责需求拆分、任务排期、验收标准、风险跟踪。
- Frontend Agent：负责门户页面、CMS 后台页面、搜索、课程展示、主题切换等前端功能。
- Backend Agent：负责接口、数据库、认证、CMS 配置、课程检索、用户注册登录等后端功能。
- QA Agent：负责功能验收、接口测试、页面测试、回归测试、投标功能点覆盖检查。

## 项目记忆文件

所有 Agent 在开始工作前必须先阅读：

1. PROJECT_MEMORY.md
2. TASKS.md
3. 当前目录下的 AGENTS.md

## 协作规则

- 不要只依赖聊天上下文。
- 重要结论必须写入 PROJECT_MEMORY.md。
- 已完成任务必须更新 TASKS.md。
- 新增接口必须同步到 docs/API.md。
- 前后端字段变更必须同步更新接口文档。
- 完成功能后必须给出验证命令或测试方式。

## 当前优先级

优先完成教学门户模块 13 项功能中的可运行闭环，包括：
门户展示、CMS 配置、登录认证配置、课程检索、课程模块展示、新闻公告、推荐课程、新课速递、名师风采、友情链接。
