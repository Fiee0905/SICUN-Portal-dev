# 注册登录、权限区分与旧版 CMS 清理专项范围

更新日期：2026-05-06  
责任角色：PM Agent  
范围：围绕“注册登录和权限区分最小闭环 + 删除旧版 CMS”的 PM 视角任务边界、验收标准、风险跟踪。

## 专项目标

在不扩大业务范围的前提下，优先形成一个可联调、可验收的最小闭环：

1. 校外用户可注册、登录、退出，并能获取当前用户信息。
2. 至少区分游客、普通登录用户、管理员三类访问身份。
3. 管理后台和 CMS 配置入口必须要求管理员身份，普通用户和游客不能访问。
4. 门户公开页面、课程检索、新闻公告等可匿名浏览的功能不因鉴权接入被破坏。
5. 旧版 CMS 入口、路由、菜单和文档口径完成清理或迁移说明，最终只保留一套后台入口和一套验收路径。

## 一期范围

| 范围项 | PM 口径 | 责任建议 | 验收证据 |
| --- | --- | --- | --- |
| 注册 | 支持校外用户提交账号、密码和基础资料；一期可不包含短信/邮箱验证码，但必须明确用户初始状态。 | Frontend / Backend | 注册页面路径、注册接口返回、数据库用户状态。 |
| 登录 | 支持账号密码登录，返回登录态或 Token；失败时有明确错误提示。 | Frontend / Backend | 登录页面路径、登录接口示例、错误账号/密码用例。 |
| 退出 | 支持主动退出并清理前端用户态；退出后访问后台应回到登录或无权限状态。 | Frontend / Backend | 退出按钮路径、退出接口或本地状态清理说明。 |
| 当前用户 | 登录后可获取当前用户资料、角色和权限标识。 | Backend / Frontend | `/users/me` 或等价接口，页面用户态展示。 |
| 权限区分 | 最小区分 `游客`、`普通用户`、`管理员`；后台配置和 CMS 管理仅管理员可访问。 | Backend / Frontend / QA | 后台接口 401/403 用例，前端路由守卫或菜单隐藏用例。 |
| 公开访问 | 门户首页、课程检索、公告新闻、友情链接等公开门户能力保持匿名可访问。 | Frontend / QA | 匿名浏览页面冒烟记录。 |
| 旧版 CMS 清理 | 删除或下线旧版 CMS 菜单/路由/入口，避免 `/cms/*` 与 `/admin/*` 双后台并存造成验收口径混乱；如暂不能删除，必须给出迁移映射和停用说明。 | Frontend / Backend / PM | 路由清单、菜单截图或页面路径验证、文档说明。 |

## 暂不纳入一期

- 真实学校统一身份认证完整接入，先保留为 P09 外部依赖，允许 Backend 提供可配置/Mock 闭环。
- 短信验证码、邮箱验证、找回密码、安全问题等增强能力，除非投标文件明确要求一期必须演示。
- 精细到按钮级、字段级的数据权限；一期只要求角色级访问控制和后台入口保护。
- 多租户、组织架构、复杂审批流。

## 验收标准

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| A01 | 游客访问公开门户 | 未登录状态可访问首页、课程检索、新闻公告和友情链接；不得因鉴权接入出现 401/403 或空白页。 |
| A02 | 游客访问后台 | 未登录访问后台首页、CMS 配置、页面配置、站点配置等管理路径时，被引导登录或返回 401/403。 |
| A03 | 普通用户访问后台 | 普通用户登录后可访问允许的个人/课程资源，但访问后台管理接口或页面时返回 403 或展示无权限状态。 |
| A04 | 管理员访问后台 | 管理员登录后可进入统一后台入口，完成 CMS/页面配置相关管理操作。 |
| A05 | 注册登录退出闭环 | 注册成功、登录成功、获取当前用户、退出成功四步可连续执行；失败场景有明确提示。 |
| A06 | 权限接口一致性 | 前端路由、后端接口、接口文档中的角色名称和权限语义一致。 |
| A07 | 旧版 CMS 清理 | 验收入口只保留统一后台路径；旧版 CMS 菜单/路由不可被误用，或已在文档中标记为废弃并给出迁移路径。 |
| A08 | 回归不破坏 P01/P07/P08/P11/P12/P13 | 接入认证后，门户展示、模块隐藏、主题配置、课程检索、预设关键词和内容模块仍可按既有验收方式通过。 |

## 风险与处理

| 风险 | 影响 | 处理建议 |
| --- | --- | --- |
| 真实统一身份认证资料未确认 | P09 无法完成真实校内登录，只能先验收 Mock/配置化闭环。 | PM 继续跟踪 CAS/OAuth 协议、回调地址、测试账号和字段映射；一期不阻塞校外注册登录闭环。 |
| 后端只做前端菜单隐藏而未做接口鉴权 | 普通用户可绕过页面直接调用后台接口，验收风险高。 | 后端必须以接口 401/403 作为验收依据，前端守卫只作为体验增强。 |
| 旧版 CMS 与新版 Admin 并存 | 演示和验收路径混乱，可能配置写入不同数据源。 | 一期要求统一入口；无法删除时必须标废弃、隐藏菜单，并给出路由迁移映射。 |
| 角色命名不一致 | 前端、后端、QA 用例无法对齐。 | 统一使用游客、普通用户、管理员三类口径，并在接口文档同步英文枚举。 |
| 接入鉴权破坏公开门户 | 原本匿名可访问的门户页面出现登录拦截。 | QA 必须把匿名门户冒烟列为回归必测项。 |

## 验证方式建议

- 前端页面：匿名访问首页/课程检索；普通用户登录后访问后台；管理员登录后访问后台；退出后再次访问后台。
- 后端接口：分别用无 Token、普通用户 Token、管理员 Token 请求管理接口，断言 401/403/200。
- 文档检查：`docs/API.md` 与 `docs/api-spec.md` 中认证接口、角色字段、错误码、后台权限说明一致。
- 旧版 CMS 检查：确认导航菜单、路由表、页面跳转入口和验收文档不再指向废弃 CMS。
